# codyssey-m2

## 1. 프로젝트 개요
이 프로젝트는 Python으로 구현한 콘솔 기반 퀴즈 게임입니다.  
사용자는 메뉴를 통해 퀴즈를 풀고, 새로운 퀴즈를 추가하고, 등록된 퀴즈 목록과 최고 점수 및 게임 기록을 확인할 수 있습니다.  
또한 프로그램 종료 후에도 퀴즈 데이터와 점수 정보가 유지되도록 `state.json` 파일에 데이터를 저장하고 다시 불러오도록 구현했습니다.

이 프로젝트는 Python 기본 문법, 클래스 설계, 파일 입출력(JSON), 예외 처리, Git 브랜치 및 병합 흐름을 직접 경험하는 것을 목표로 했습니다.

---

## 2. 퀴즈 주제와 선정 이유
퀴즈 주제는 **넌센스 퀴즈**입니다.

예를 들어 다음과 같은 문제가 포함되어 있습니다.

- 우유가 넘어지면?
- 왕이 넘어지면?
- 바나나가 웃으면?

이 주제를 선택한 이유는 다음과 같습니다.

- 처음 만드는 콘솔 게임인 만큼 문제를 직관적으로 이해할 수 있어야 했습니다.
- 넌센스 퀴즈는 짧고 재미있어서 직접 문제와 선택지를 만들기 좋습니다.

---

## 3. 실행 방법

### 실행 환경
- Python 3.10 이상
- 외부 라이브러리 없음 (표준 라이브러리만 사용)

### 실행 명령어
프로젝트 루트에서 아래 명령어를 실행합니다.

```bash
python3 main.py
```

### 프로그램 실행 흐름
1. 메뉴가 출력됩니다.
2. 번호를 입력해 원하는 기능을 선택합니다.
3. 퀴즈 풀기, 퀴즈 추가, 퀴즈 목록, 점수 확인, 종료 기능을 사용할 수 있습니다.
4. 각 기능에서 `Ctrl + c`를 누르면 해당 기능을 중단하고 다시 메뉴 화면으로 돌아갈 수 있습니다.

## 4. 기능 목록

### 4-1. 메뉴 기능
- 프로그램 실행 시 메뉴 출력
- 번호 입력으로 기능 선택

### 4-2. 퀴즈 풀기
- 저장된 퀴즈 중에서 문제를 출제
- 출제할 문제 수를 선택 가능
- 문제 순서를 랜덤하게 섞어 출제
- 각 문제마다 힌트를 볼지 y/n으로 선택 가능
- 정답/오답 여부 출력
- 모든 문제 풀이 후 결과 출력

### 4-3. 힌트 기능
- 모든 문제는 힌트를 가짐
- 문제를 풀 때 힌트를 볼지 선택 가능
- 힌트 사용 여부에 따라 점수 차등 적용

### 4-4. 점수 계산
- 틀리면 0점
- 힌트를 보고 맞히면 1점
- 힌트를 보지 않고 맞히면 2점
- 게임 결과는 실제 획득 점수 기준으로 계산

### 4-5. 퀴즈 추가
- 새로운 문제 등록 가능
- 문제, 선택지 4개, 정답 번호, 힌트를 입력받아 저장
- 잘못된 입력에 대한 검증 처리 포함

### 4-6. 퀴즈 목록 확인
- 현재 저장된 퀴즈 목록 확인 가능
- 등록된 퀴즈가 없을 경우 안내 메시지 출력

### 4-7. 최고 점수 확인
- 가장 좋은 게임 기록 확인 가능
- 게임 결과와 기존 최고 기록을 비교하여 갱신

### 4-8. 점수 기록 히스토리
- 최고 점수뿐 아니라 모든 게임 기록 저장
- 게임 시작 시간, 푼 문제 수, 맞힌 문제 수, 획득 점수 기록

### 4-9. 파일 저장/불러오기
- 프로그램 종료 후에도 데이터 유지
- 퀴즈, 최고 점수, 게임 히스토리를 `state.json`에 저장
- 파일이 없거나 손상되었거나 읽기/쓰기 오류가 발생하면 기본 퀴즈 생성

## 5. 파일 구조
```
codyssey-m2/
├── __pycache__/
├── docs/
│   └── images/
│   └── environment.md
│   └──log_screenshot.md
│   └── running_screen.md
├── .gitignore
├── main.py
├── quiz_game.py
├── quiz.py
├── README.md
└── state.json
```

### 파일 설명
- `main.py`: 프로그램 시작 파일입니다. `QuizGame` 객체를 생성하고 실행합니다.
- `quiz_game.py`: `QuizGame` 클래스가 정의된 파일입니다.
메뉴 출력, 퀴즈 진행, 퀴즈 추가, 점수 계산, 파일 저장/불러오기 등 게임 전체 흐름을 관리합니다.
- `quiz.py`: `Quiz` 클래스가 정의된 파일입니다.
문제, 선택지, 정답, 힌트 데이터를 관리하고 퀴즈 출력, 정답 확인, 점수 계산에 사용됩니다.
- `state.json`: 프로그램 실행 중 생성/사용되는 데이터 파일입니다.
퀴즈 목록, 최고 점수, 게임 히스토리를 저장합니다.
- `README.md`: 프로젝트 설명 문서입니다.
- `docs`: 프로그램 실행 화면, 로그 기록 등의 문서와 이미지들이 묶여있습니다.
- `images`: 문서에 들어가는 이미지들이 모여있습니다.
- `environment.md`: 개발 환경 설정 스크린샷 파일입니다.
- `log_screenshot.md`: branch 병합 결과 로그 파일입니다.
- `running_screen.md`: 프로그램 실행 결과 스크린샷입니다.

## 6. 데이터 파일 설명 (state.json 경로 / 역할 / 스키마)

### 경로
- 프로젝트 루트 경로의 `state.json`

### 역할
- 퀴즈 데이터를 저장합니다.
- 최고 점수 정보를 저장합니다.
- 게임 플레이 히스토리를 저장합니다.
- 프로그램을 다시 실행해도 이전 상태를 유지할 수 있도록 합니다.

### 스키마 예시
```
{
    "quizzes": [
        {
            "question": "우유가 넘어지면?",
            "choices": ["밀크콩", "초코우유", "아야", "커피"],
            "answer": 3,
            "hint": "다쳤을 때 나는 소리를 떠올려보세요."
        }
    ],
    "best_record": {
        "score": 7,
        "total_count": 5,
        "best_count": 4
    },
    "history": [
        {
            "played_at": "2026-04-09 21:30:00",
            "score": 7,
            "total_count": 5,
            "correct_count": 4
        }
    ]
}
```

### 필드 설명
- `quizzes`: 저장된 퀴즈 목록입니다.
- `question`: 문제 내용입니다.
- `choices`: 4개의 선택지 목록입니다.
- `answer`: 정답 번호입니다. 1 ~ 4 범위를 사용합니다.
- `hint`: 문제 힌트입니다.
- `best_record`: 최고 점수 기록입니다.
- `best_record.score`: 최고 기록의 실제 획득 점수입니다.
- `best_record.total_count`: 최고 기록에서 푼 문제 수입니다.
- `best_record.best_count`: 최고 기록에서 맞힌 문제 수입니다.
- `history`: 모든 게임 기록 목록입니다.
- `history.played_at`: 게임 시작 시간입니다.
- `history.score`: 해당 게임에서 획득한 실제 점수입니다.
- `history.total_count`: 해당 게임에서 푼 문제 수입니다.
- `history.correct_count`: 해당 게임에서 맞힌 문제 수입니다.
